# Linux_易混易错知识点总结

## 第一章 Linux基础

###### 1、Linux系统的内核版本的命名方式为:Linux r.x.y,其中数字x表示( )

**偶数稳定**

###### 2、系统调用与库函数( )

**系统调用工作在OS内核态，库函数工作是OS用户态。**

###### 3、将/home/test文件的拥有者修改为tom,命令为( )

chown tom /home/test

ps:这里的参数设置就和cp，mv等命令不太一致

###### 4、将文件所属组修改为student,命令为( )

chgrp student /home/tes

ps：用户组的操作，确实是和文件操作参数设置有区别

###### 5、对于终止不了的进程,可以利用选项( )进行终止。

kill -9

ps：其功能就是“消灭进程”

###### 6、现进行Linux环境下的C应用程序开发,想利用mkdir系统调用,但不知道如何用,可以利用如下帮助获取用法( )

man 2 mkdir

ps：man命令的2这个参数，是查看相关命令的详细用法

###### 7、重启网卡配置使用(  )

/etc/init.d/network restart

###### 8、需要一次创建多级目录,采用( )

mkdir –p 目录

###### 9、需要复制目录,采用(  )

cp -r 源目录 目的目录

## 第二章 Linux系统开发环境

###### 1、Gcc编译器在链接程序时,使用的命令是( )

ld

ps:ld是链接命令

###### 2、Gcc编译器在编译过程中,调用cc1进行编译时,生成的文件是( )

s的汇编源码文件

###### 3、执行make时,假如makefile文件存在,则以下哪个一定会执行编译( )

目标文件不存在

ps：make是根据时间戳编译文件，已经修改或者不存在的，就会进行编译。

###### 4、Makefile文件自动化变量中,表示目标文件名称的是( )

 $@

###### 5、Makefile文件自动化变量中,表示所有依赖文件的是( ) 

$^

###### 6、Makefile文件自动化变量中,表示第一个依赖文件的是( )

$<

###### 7、Linux的静态库、共享库,都可由( )文件创建

.o文件

ps：编译后生成.o文件

###### 8、Linux的静态库、共享库,都可由( )文件创建

大

ps：如果有较大和大来选，直接选大

###### 9、编译时,Linux的动态库( )合并入可执行文件

会

ps：这就是用动态库的程序体量比用静态库小的原因

###### 10、Linux优先选择( )进行编译

动态库

ps：一般都是放在/usr/lib OS会直接进行查询

###### 11、以下哪个可以生成静态库libm.a( )

ar -rc libm.a m.o

###### 12、在共享库的创建过程中,获取目标文件在编译时必须加上的选项是( )

 -fpic
